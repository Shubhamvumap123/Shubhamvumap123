## 2024-05-18 - Accessibility of Dynamic SVGs and Aligning Multiple Badges
**Learning:** Dynamic graphics (like `capsule-render` SVGs or decorative GIFs) used in profile READMEs lack inherent semantic descriptions for screen readers, unlike text elements. Additionally, when centering a list of Markdown badges, wrapping them in `<p align="center">` combined with converting them to standard HTML `<img>` tags ensures they correctly flow inline while being centered as a single block.
**Action:** When working on GitHub profile profiles, explicitly add `alt` text to dynamic/decorative elements (e.g., `<img ... alt="Footer wave graphic">`). Furthermore, to achieve a responsive, centered grid of badges, convert Markdown badge syntax (`![alt](url)`) to HTML `<img>` tags enclosed in a `<p align="center">` block.
